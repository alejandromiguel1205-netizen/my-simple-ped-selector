Overview

Welcome to the advanced developer documentation portal for My Simple Ped Selector. This desktop tool functions as an efficient compiler for custom Grand Theft Auto V character files, optimizing the installation phase and cutting the security risk factor down to just 5% compared to manual XML modifications.

The system utilizes a custom architecture capable of generating individual data matrices, allowing modders to register models without repackaging heavy game components every time.

OpenIV Integration

To successfully link the software compiler logic with the RAGE engine architecture, you must match your physical asset directories inside OpenIV's streaming file container (peds.rpf) according to the format requirements:

Standard Mode (Non-Streamed)

Applicable if the asset package contains exactly 4 separate core engine fragments: .ydd (geometry), .ytd (textures), .yft (physics), and .ymt (meta metadata). These must sit loose inside the root directory of your mod archive.

Complex Mode (Streamed)

Required for multi-texture assets or clothing fragments. You must build a dedicated subfolder named exactly like your target model inside peds.rpf, and load the texture asset clips directly inside that sub-directory partition.

Application Configuration

The deployment pipeline requires exactly three phases to compile character properties safely from the interface to the local engine system:

1

Profile Mapping

Trigger creation with Ctrl + N, assign the model string name (case-sensitive mapping), match the gender entity and toggle the streaming checkbox.

2

Rebuild Phase

Execute the REBUILD! action button. The custom progress thread compiles and indexes rows directly to protect your database assets.

3

Secure Commit

Press Ctrl + G before exiting to permanently serialize settings and synchronize configurations with your system environment.

Troubleshooting

Modding environmental platforms requires specific configuration adjustments. Review these protocol responses if runtime issues occur:

Instant Desktop Crash (CTD) on Entity Spawning

This runtime error surfaces when you exceed the stock allocated memory array blocks of GTA V. To solve this restriction, deploy an upgraded custom gameconfig.xml file, complemented by HeapAdjuster and Packfile Limit Adjuster plugins in the root game path.

Entity Renders Invisible or with Broken Texture Shading

Points directly to a structural mismatch. Check your Is Streamed toggle values. If your asset requires a subfolder architecture within OpenIV but the compiler flagged it as False, the engine shader pipeline will drop rendering layers, masking the entity or breaking textures.
