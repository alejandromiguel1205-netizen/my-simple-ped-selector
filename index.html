Welcome!

Welcome to the official My Simple Ped Selector Help Center. This interface was engineered to guide you through adding custom addon characters (Peds) to GTA V seamlessly, cutting out manual XML editing completely and preventing file corruption.

Follow the structured sections below to map out your files in OpenIV and register them correctly inside the application framework.

Download on GTA5-Mods

1. OpenIV Directory Setup

To begin, open OpenIV, ensure Edit Mode is fully active, and navigate into your custom addon DLC package partition, specifically inside the peds.rpf file container. Verify your loose file patterns:

Option A: Non-Streamed (Standard)

Select this route if your character bundle consists strictly of 4 loose files with these matching extensions:

ydd, ytd, yft, ymt

Action: Drag and drop all 4 fragments directly into the root level of your peds.rpf archive layout.

Option B: Streamed (Complex)

Select this route if your mod pack has dedicated folders for high-definition texture asset packages or geometry layers.

Action: Right-click inside peds.rpf, create a new folder, and match its name exactly to your character model string. Open it, and drop all asset sheets inside.

2. Application Registration

Once your file directories match in OpenIV, open up the My Simple Ped Selector client workspace and execute these 3 core integration steps:

1

Initialize Profile Matrix

Navigate to the top workspace layout toolbar and click Ped -> Create New Ped (or execute the hotkey combo Ctrl + N). Populate the required parameters:

  • Ped Name: Input the exact string name of the files loaded into OpenIV (Case-sensitive mapping).
  • Ped Gender: Map the bone animation base to Male or Female.
  • Is Streamed: Set to False if you implemented Option A (loose entries), or set to True if you followed Option B (subfolder setup).

Click Save Ped to add the row to the active interface index.

2

Execute Database Rebuild

Locate the upper tool options and trigger the REBUILD! action button. The green compiler progress bar will cycle, loading row properties securely into the local database index. Wait for completion.

3

Lock Progress Serialization

Commit your configuration data to the system disk by selecting File -> Save or executing the hotkey combo Ctrl + G. If an asterisk symbol (*) marks the window head title, your workspace contains uncommitted changes.

Hotkey Commands

Utilize these native hotkey controls inside the interface environment to expedite data profile management:

Action Shortcut System Operation
Create Profile Ctrl + N Instantly prompts the creation dialog panel.
Save Workspace Ctrl + G Serializes and commits all current rows into the local database.
Export As Profile Ctrl + A Saves the selected row out into a single shareable .spm file.
Import External Profile Ctrl + O Parses an external .spm dataset package straight into the active grid.
Duplicate Entry Ctrl + D Clones data using protective routing suffixes (e.g., -copy, -2).
Purge Profile Delete Drops the selected matrix entry after running a safety validation prompt.

Environmental Troubleshooting

Review these environment fix logs if you experience loading discrepancies or deployment anomalies during runtime:

Game Engine Crashes Instantly (CTD) Upon Entity Spawning

This crash occurs when you breach Grand Theft Auto V's default internal memory allocation limits for mod assets. To solve this, deploy an updated custom gameconfig.xml matched with HeapAdjuster and Packfile Limit Adjuster plugins inside your root directory.

Character Model Spawns Invisible or With Corrupted Textures

Points to a streaming structural mismatch. Cross-check your Is Streamed option. If your mod files require a directory layout inside OpenIV but you flagged it as False inside the compiler, the engine won't link fragment dependencies, rendering the model out or causing massive alpha textures clipping.
