How to Add & Configure Custom Peds

  • 1

    OpenIV File Management

    First, inject your modified character files inside your addon DLC pack's streaming archive folder (peds.rpf) depending on its type:

    Non-Streamed Peds (Standard)

    If your ped has exactly 4 core files (.ydd, .ytd, .yft, .ymt), place them directly inside the root of peds.rpf.

    Streamed Peds (Complex)

    If your character features loose high-definition texture fragments, you must create a subfolder named exactly like the model inside peds.rpf and place all files inside it.

  • 2

    Application Profile Setup

    Launch the app and map out your character parameters into the editor interface:

    • Press Ctrl + N to prompt the creation form.
    • Ped Name: Type the exact name of the files in OpenIV (Case-sensitive).
    • Ped Gender: Assign if the model base behaves as Male or Female.
    • Is Streamed: Toggle to True if you created a dedicated subfolder in OpenIV, or False if it's a single standard model.
  • 3

    Database Compilation & Save

    Finalize the mod integration into the game engine assets:

    • Click the prominent REBUILD! button in the main panel. The green progress bar will animate while loading entries into the game index.
    • Once the success notification appears, press Ctrl + G to permanently lock in your database updates.

Technical Troubleshooting (FAQ)

The game crashes instantly to desktop when spawning characters

This is a standard memory limitation crash caused by exceeding Grand Theft Auto V's default asset configuration array allocations. To fully solve this, make sure you have an updated custom gameconfig.xml matched with HeapAdjuster and Packfile Limit Adjuster installed directly into your main GTA V game directory folder.

The custom character spawns completely invisible or with broken glitch textures

This points to an asset streaming type mismatch. Double-check the Is Streamed option inside the editor list. If your character files require a folder structure in OpenIV but you marked it as False in the selector app, the game engine won't parse the fragment dependencies, causing models to clip out or turn invisible.
